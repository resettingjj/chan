Unity로 Flock 알고리즘 구현

My project (1)




유니티를 처음 사용해보며, C#기초를 다지고 유니티의 기능들을 알기 위해 처음 제작한 프로젝트인 Flocking Algorithm(군집 알고리즘)이다.
오브젝트를 조종하는 기본적인 방법과 유니티 엔진의 작동 방식, 오브젝트에 코드를 삽입하여 작동시키는 방법, 코드를 묶고 클래스를 받아오는 방법 등을 활용하는 법,
유니티의 에디터 기능을 사용하여 버튼을 직접 만들고 유니티 개발 환경을 바꾸는 법 등을 배웠다.

군집 알고리즘은 물고기나 새 등 떼를 지어 이동하는 자연 속 집단의 행동에서 영감을 받은 알고리즘으로, 정렬, 응집, 분리의 3가지 알고리즘에 의해 실행되는 동작 알고리즘이다.
이 세 알고리즘은 주변 개체의 방향과 같은 방향으로 가려고 하는 정렬 알고리즘, 주변 개체들의 중심으로 가려고 하는 응집알고리즘, 주변 개체들과 일정한 거리를 두려고 하는 분리 알고리즘으로 나뉜다.
 해당 알고리즘을 unity2D엔진을 사용하여 구현하기로 마음먹었고, 2차원에서 움직임을 담당하는 Vector2를 사용하여 구현하기 시작했다.
우선 세 알고리즘 중 정렬 알고리즘을 먼저 구현하기로 마음먹었는데, 그럴려면 주변 이웃 개체를 만들고 각 객체에 모두 알고리즘을 할당해 주어야 했다.
따라서 해당 알고리즘을 만들기 위해서 가장 먼저 필요한 것은 Instantiate함수를 사용하여 객체의 복제본을 우선적으로 만들어주는 것이었다.
이 상태에서 복제본들끼리 좌표리스트를 주고받기 위해 이동클래스를 따로 만들어 서로 정보를 주고받게 해야 했다.
이후에도 주변 개체의 이동속도에 맞추고, 회피움직임과 응집움직임을 만드는 과정도 남아있었고, 무척이나 까다로웠다.

게다가 이렇게 만들어진걸 실제로 유니티에서 적용하기 위해선, 이 코드들을 단 하나로 합쳐야만 했다.
public을 이용한 통합움직임 코드를 만들어 앞서 말한 코드들의 리턴값을 이 코드로 받아왔고, 받아온 리턴값을 모두 더한 움직임을 만들어 낼 수 있었다..

단순한 세가지 알고리즘으로 만들어진 이 군집은 새 떼나 물고기 떼등을 시뮬레이션 하거나, 박쥐 떼가 날아다니는 영상,
소셜 네트워크에서 의견의 흐름을 시뮬레이션해서 미래 의견을 예측하거나 분산 시스템등에서 이용되는 등 여러 방면으로 사용되는 알고리즘이다.

이 알고리즘을 좀 더 알고싶어, 무리들이 회피하는 포식자(공)를 만들었을 때 무리들이 어떤 영향을 주고받는지 확인하기 위해 다시 프로그래밍을 시작했다.
우선, 직접 조종 가능한 공을 구현하기 위해 OnMove를 이용하여 직접 조종할 수 있게 만든 후, Transform으로 공의 좌표를 받아와 객체 사이의 거리를 이용해 객체의 방향을 결정하거나 Destroy 하는 등의 알고리즘을 만들었다.
그 결과 한 무리 또는 단일 객체가 공을 피해 움직이는것과 같은 특정 행동을 했을 때 주변 무리들에 영향을 주는 Flock알고리즘의 흥미로운 점을 관찰할 수 있었다.
